# 同步工具

## Seamphore

操作系统里信号量的作用，用来控制线程的同步

构造器初始化固定数量的state表示资源数量

**1.acquire（n）/acquire（）:**  

表示获取多少信号，state-n，如果state<0,    **构造节点进入AQS的同步队列等待条件满足(自旋tryAcquireShared())唤醒**

```java
//acquire(n) 调用AQS的
public final void acquireSharedInterruptibly(int arg)
            throws InterruptedException {
    //小于0代表资源不足，
    if (tryAcquireShared(arg) < 0)
            //构造节点进入AQS的同步队列,期间不断调用tryAcquireShared，直到资源足够
            doAcquireSharedInterruptibly(arg);
}

//自己实现tryAcquireShared
 protected int tryAcquireShared(int acquires) {
             for (;;) {
                int available = getState();
                int remaining = available - acquires;
                if (remaining < 0 ||
                    compareAndSetState(available, remaining))
                    return remaining;
            }
}
```



**2.release（n）/release（）:** 

每次release表示释放一个信号，state+n，**同步队列唤醒一个线程**

```java
//release(n) 调用AQS的releaseShared(arg)
public final boolean releaseShared(int arg) {
        if (tryReleaseShared(arg)) {
            //唤醒同步队列的一个节点
            doReleaseShared();
            return true;
        }
        return false;
    }

//自己实现tryReleaseShared
 protected final boolean tryReleaseShared(int releases) {
            for (;;) {
                //判断是否有锁再等了，非公平没有这一步
                if (hasQueuedPredecessors())
                    return -1;
                //简单加state
                int current = getState();
                int next = current + releases;
                //释放表示增加资源
                if (compareAndSetState(current, next))
                    return true;
            }
        }
```





## CountDownLatch

用来实现一个任务必须等到一批任务完成后才能执行

构造器初始化固定数量的state，表示等待的线程数量

await()：

```java
//acquire(n) 调用AQS的
public final void acquireSharedInterruptibly(int arg)
            throws InterruptedException {
    //小于0代表资源不足，
    if (tryAcquireShared(arg) < 0)
            //构造节点进入AQS的同步队列,期间不断调用tryAcquireShared，直到资源足够
            doAcquireSharedInterruptibly(arg);
}

//自己实现tryAcquireShared
 protected int tryAcquireShared(int acquires) {
     //等待为0表示成功，返回正数
       return (getState() == 0) ? 1 : -1;
}
```

```java
public final boolean releaseShared(int arg) {

            //唤醒同步队列的一个节点
            doReleaseShared();
            return true;
        }
        return false;

 protected final boolean tryReleaseShared(int releases) {
            for (;;) {
                int c = getState();
                if (c == 0)
                    return false;
                //简单的减去1，表示等待线程少一个
                int nextc = c-1;
                if (compareAndSetState(c, nextc))
                    return nextc == 0;
            }
        }
```

## CyclicBarrier(没有使用AQS)

用来实现多个任务同时执行

由于CyclicBarrier可以重复使用因此使用Generation来代表使用的轮次

```java
private static class Generation {
        boolean broken = false;
    }
```



await()

```java
//CyclicBarrier await()调用dowait()
private int dowait(boolean timed, long nanos)
        throws InterruptedException, BrokenBarrierException,
               TimeoutException {
        final ReentrantLock lock = this.lock;
        lock.lock();
        try {
            final Generation g = generation;
            
          //意外: 如果线程中断了，抛出异常
        if (Thread.interrupted()) {
                  //如果当前线程被中断会做以下三件事
                  //1.打翻当前栅栏
                  //2.唤醒拦截的所有线程
                  //3.抛出中断异常
            breakBarrier();
            throw new InterruptedException();
        }
			
            //
            int index = --count减小到0，开始运行;
            //count减小到0，开始运行
            if (index == 0) {  // tripped
                boolean ranAction = false;                               
                ranAction = true;
                //这个方法会调用signalAll()，唤醒所有在条件队列的线程
                nextGeneration();
                return 0;
               
            }

            // loop until tripped, broken, interrupted, or timed out
            for (;;) {
                try {
                    //卡在这里循环等待
                   trip.await();                   
                } catch (InterruptedException ie) {
                    //中断也唤醒，若当前线程在等待期间被中断则打翻栅栏唤醒其他线程
                    if (g == generation && ! g.broken) {
                        breakBarrier();
                        throw ie;
                    } else {
                        // We're about to finish waiting even if we had not
                        // been interrupted, so this interrupt is deemed to
                        // "belong" to subsequent execution.
                        Thread.currentThread().interrupt();
                    }
                }
                //当所有线程到达正常唤醒
                
                
				//意外: 当有任何一个线程中断了，就会调用breakBarrier方法
                if (g.broken)
                    throw new BrokenBarrierException();
				//g != generation表示正常换代了，返回当前线程所在栅栏的下标          
                if (g != generation)
                    return index;

                if (timed && nanos <= 0L) {
                    breakBarrier();
                    throw new TimeoutException();
                }
            }
        } finally {
            lock.unlock();
        }
    }
```
